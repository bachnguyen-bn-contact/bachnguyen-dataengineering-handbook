# Security & governance

## Workspace roles [Basics]

Access starts at the workspace: Admin, Member, Contributor and Viewer roles control who can build, run and read. Grant the least privilege that lets someone do their job.

## Fine-grained security [Advanced]

Within items you can apply **row-level**, **column-level** and **object-level** security, plus **dynamic data masking**. OneLake security and **sensitivity labels** extend protection to the data itself.

> [!CAUTION]
> Test security as the end user, not as an admin. Admins often bypass the very rules you are trying to verify.
