# Study plan & advice

A realistic plan for someone with some data background, putting in **6–8 hours a week**. Stretch the Spark and KQL weeks if either is new to you.

## A 6-week outline

| Week | Focus |
|---|---|
| 1 | Read the study guide end to end. Build your first lakehouse; do ingestion + basic transforms. Take the practice assessment **once** to baseline. |
| 2 | **Ingest & transform** — pipelines, Dataflows Gen2, full vs. incremental loads. |
| 3 | **Spark** — PySpark notebooks, Spark SQL, the medallion flow Bronze → Silver → Gold. |
| 4 | **Real-Time Intelligence & KQL** — Eventstreams, Eventhouse, querying data in motion. |
| 5 | **Manage the solution** — workspaces, security/governance, Git integration, deployment pipelines. |
| 6 | **Monitor & optimise** — run monitoring, troubleshooting, performance tuning. Retake the practice assessment; review every miss. |

## Advice I'd give my past self

- **KQL is the sleeper topic.** If you come from a SQL-only or other-cloud background, it's the biggest genuinely new skill. Give it dedicated time rather than cramming it last.
- **Don't skip a domain.** All three are weighted roughly the same, so a weak domain costs you as much as any other.
- **Learn the docs layout, not just the content.** You'll have Microsoft Learn open during the exam — knowing *where* things live turns lookups into seconds.
- **Treat the skills list as a checklist.** Every bullet is a potential question. Tick each one only after you've done it hands-on, not just read about it.
- **Map preview vs. GA.** Fabric moves fast and the exam can include commonly used preview features, so don't assume a feature is out of scope just because it's new.

## Pre-exam checklist

- [ ] Every bullet of the study guide ticked off, hands-on
- [ ] Practice assessment comfortably passed, twice
- [ ] Built a lakehouse, a pipeline, a Dataflow Gen2 and a KQL query yourself
- [ ] Comfortable navigating the Fabric docs quickly
- [ ] Reviewed security, Git integration and deployment pipelines specifically
